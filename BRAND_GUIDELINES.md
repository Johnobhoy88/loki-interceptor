- **Intelligent**: Our AI-powered system provides smart, context-aware compliance checks
- **Precise**: We deliver accurate, detailed compliance analysis with zero ambiguity

### Target Audience

- Enterprise compliance officers
- Legal teams
- Financial institutions
- HR departments
- Regulatory affairs professionals

- **Light (300)**: Reserved for special cases
- **Normal (400)**: Body text
- **Medium (500)**: Emphasized text, navigation
- **Semibold (600)**: Headings, important labels
- **Bold (700)**: Strong emphasis, warnings

### Typography Guidelines

**Headings**
- Use semibold weight (600)
- Maintain clear hierarchy (3XL ‚Üí 2XL ‚Üí XL)
- Keep letter-spacing slightly tight (-0.5px to -0.25px)

**Body Text**
- Use base size (14px) for most content
- Line height: 1.5 for readability
- Maximum line length: 65-75 characters

**Labels & UI Text**
- Use uppercase for labels (with wider letter-spacing)
- Use medium weight (500) for buttons and navigation
- Keep button text concise and action-oriented

## Voice & Tone

### Brand Voice

**Clear & Direct**
- Use plain language, avoid jargon
- Be concise and get to the point
- Example: "Document failed validation" not "The validation process has encountered an error"

**Professional & Authoritative**
- Demonstrate expertise in compliance
- Provide confident guidance
- Example: "GDPR requires explicit consent" not "We think GDPR might need consent"

**Helpful & Supportive**
- Guide users through complex compliance issues
- Offer solutions, not just problems
- Example: "Add the data retention policy to proceed" not "Missing data retention policy"

### Tone Variations

**Success Messages**
- Positive and congratulatory
- Example: "Compliance validated successfully! All checks passed."

**Error Messages**
- Clear and actionable
- Explain what went wrong and how to fix it
- Example: "GDPR violation detected: Missing consent clause. Add explicit consent language to section 2."

**Warning Messages**
- Cautionary but not alarming
- Provide context and next steps
- Example: "This document may require FCA review. Consider adding disclosure statements."

**Informational Messages**
- Neutral and helpful
- Provide context without judgment
- Example: "Tax regulations updated. Review your documents for compliance."

### Writing Guidelines

**Do:**
‚úÖ Use active voice ("Validate document" not "Document should be validated")
‚úÖ Be specific ("Add GDPR consent clause" not "Fix GDPR issue")
‚úÖ Use parallel structure in lists
‚úÖ Write for scanning (clear headings, short paragraphs)

**Don't:**
‚ùå Use all caps (except for acronyms: GDPR, FCA)
‚ùå Use exclamation points excessively
‚ùå Use humor in error messages
‚ùå Blame the user ("You failed to..." ‚Üí "Document missing...")

## Visual Style

### Design Principles

1. **Clarity First**
   - Information hierarchy is paramount
   - Remove unnecessary decoration
   - Every element serves a purpose

2. **Professional Polish**
   - Consistent spacing and alignment
   - Subtle shadows and borders
   - Refined micro-interactions

3. **Data Transparency**
   - Make complex compliance data understandable
   - Use visual cues (colors, icons) to aid comprehension
   - Provide context and explanations

4. **Accessible by Default**
   - High contrast ratios
   - Clear focus indicators
   - Keyboard navigation support

### UI Patterns

**Cards & Panels**
- Use cards to group related information
- Maintain consistent padding (20px default)
- Use subtle shadows for depth

**Forms**
- Stack form fields vertically
- Provide clear labels above inputs
- Show validation feedback immediately

**Data Display**
- Use tables for structured data
- Highlight important values with color
- Provide filters and search for large datasets

**Navigation**
- Keep primary navigation visible
- Use clear, descriptive labels
- Indicate current location

### Imagery Style

**Icons**
- Use outline-style icons (2px stroke)
- Keep icons simple and recognizable
- Size: 16px (small), 20px (medium), 24px (large)

**Illustrations** (if used)
- Minimal, line-based style
- Use brand colors sparingly
- Focus on clarity over decoration

**Data Visualizations**
- Use compliance colors consistently
- Provide clear labels and legends
- Make data accessible (not color-dependent alone)

### Motion & Animation

**Principles**
- Animations should feel natural and purposeful
- Duration: 150-300ms for most interactions
- Use easing functions (ease-out for entrances, ease-in for exits)

**Common Animations**
- Fade in: New content appearing
- Slide in: Modals, side panels
- Bounce: Attention-seeking (use sparingly)
- Pulse: Loading states

## Usage Guidelines

### Digital Applications

**Web Application**
- Dark theme by default
- Light theme available as option
- Responsive design (mobile-first)

**Marketing Website**
- Can use lighter backgrounds
- Emphasize trust and professionalism
- Feature compliance icons prominently

**Documentation**
- Use monospace font for code
- Syntax highlighting using brand colors
- Clear section hierarchy

### Printed Materials

**Business Cards**
- Logo on front, centered
- Contact information on back
- Use brand primary color

**Reports & Documents**
- Use professional templates
- Include logo in header
- Use compliance colors for section dividers

### Social Media

**Profile Images**
- Use icon-only version of logo
- Ensure visibility at small sizes

**Cover Images**
- Feature full logo
- Use brand colors
- Keep text minimal

**Post Graphics**
- Maintain brand consistency
- Use compliance colors appropriately
- Include logo on branded content
